# content

**콘텐츠/데이터 관계** 정의나 MDX·CMS 연동용 소스가 있을 경우 두는 디렉터리입니다.

## 역할

- 현재는 `relations.ts` 등 보조 스크립트/타입만 있을 수 있습니다.
- 블로그, 프로젝트 상세를 마크다운/MDX로 관리하거나 외부 데이터를 매핑할 때 여기에서 export하는 패턴으로 확장할 수 있습니다.

## 참고

- 메인 포트폴리오 문구는 `messages/`와 `lib/portfolio-data.ts`에서 관리하고 있으며, 이 폴더는 추후 콘텐츠 구조를 확장할 때 사용하는 것을 권장합니다.
